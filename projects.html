1. Smart Knob in Building Automation Applications (Bachelor Thesis)

This was my bachelor's thesis project at the Technical University of Cluj-Napoca, where I designed, built, and validated a centralized controller for a smart home. The objective was to simplify the interaction with smart devices by offering a physical, tactile interface that eliminates the need for multiple mobile applications.

Key Contributions and Technologies:

  • Mechanical Design: I created a custom, 3D-printed enclosure that allowed the display's cable to pass through the hollow shaft of the motor, ensuring 360-degree rotation without the risk of entanglement.
  • Haptic Feedback: I implemented an advanced haptic feedback system using a Brushless DC (BLDC) motor and the Field-Oriented Control (FOC) technique. This allowed for the creation of unique physical sensations, such as discrete detents for fan speed control and spring-like end-stops for adjusting volume or temperature.
  • Real-Time Firmware: The firmware was developed in C++ on an ESP32 microcontroller with a dual-core architecture. I used FreeRTOS to separate the time-critical motor control logic from other tasks (like communication and display updates), ensuring instant and uninterrupted haptic feedback.
  • Smart Home Integration: The device communicates with the Home Assistant platform using the MQTT protocol. I configured MQTT sensors and automations in Home Assistant to translate physical interactions (rotation, press) into commands for various devices, such as lights and media players.

You can find the full thesis document here: Smart Knob Thesis (PDF)

2. Home Lab Infrastructure Management

I built and manage a personal server (`home lab`) on an HP ProDesk 600 G2 mini PC. It runs Ubuntu Server and hosts a variety of services via Docker and Portainer, demonstrating practical skills in system administration and infrastructure management.

Technical Details and Applications:

  • Containerization: I implemented a Docker runtime environment, managing services like Home Assistant, Nextcloud (a self-hosted file storage service), and a MariaDB database, all running in containers.
  • Secure Networking: Remote access to the server is achieved via SSH and a Tailscale VPN, ensuring a secure connection without exposing public services to the internet.
  • Automation and Monitoring: I configured an automated watchdog to restart network services in case of connectivity loss, ensuring continuous availability. I also use Cockpit for monitoring resources (CPU, storage).

3. SmartBreeze: IR Air Conditioner Control

This IoT project was designed to remotely control an infrared (IR) air conditioner using an Android mobile application and an ESP32 microcontroller. The system mimics the functions of a traditional remote control (power toggle, temperature adjustment, fan speed control) through a wireless connection over Wi-Fi.

Technologies and Implementation:

  • Platforms: The Android application was developed with Kotlin and Jetpack Compose. The firmware for the ESP32 was written in C++ using the Arduino Framework.
  • Communication: Interaction between the mobile app and the ESP32 is done via a WebSocket connection on a local Wi-Fi network.
  • Functionality: The ESP32 receives commands from the app, parses them, and sends the corresponding IR signals to the air conditioner unit.

You can find the project documentation here: SmartBreeze Documentation (PDF)
